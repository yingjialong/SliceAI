## Claude Review Loop 结果

- 记录文件：`docs/Task-detail/claude-loop-v0.3-release-prep.md`。
- Round 1 verdict：`needs_attention`，0 critical / 1 high / 1 medium / 0 low。
- Round 2 verdict：`approve`，`findings: []`。

接受并修复：

1. **High**：长 MCP tool result 经过 `Redaction.scrub` 后变成 `<truncated:N>`，该短摘要又被回填给 LLM 的 `.tool` role message，导致 `web-search-summarize` 无法看到真实 Brave 搜索结果。修复后 UI / 日志仍使用短摘要，LLM tool message 使用 pattern-only 脱敏并保留最多 16 KiB 前缀内容。
2. **Medium**：`StdioMCPClient` 按 descriptor id 缓存旧进程，Settings 修改 command / args / env 后旧 runner 会继续运行到 idle timeout 或 App 重启。修复后 stdio session 会比较 transport / command / args / env / url，启动配置变化时 teardown 并重启。

## 代码修改逻辑

1. `Redaction.scrub(_:)` 仍保持 audit / UI 语义：先替换敏感模式，再用 `<truncated:N>` 限制短字段体积。
2. 新增 `Redaction.scrubSecrets(_:)`：只替换 Bearer、`sk-`、Authorization、Cookie 等敏感模式，不做长度截断，供模型可见的工具结果使用。
3. `AgentExecutor` 将 MCP result 分成两条路径：
   - `summarize(result:)`：给 ResultPanel / error row 的短摘要，仍最多 200 字符。
   - `toolMessageContent(result:)`：给下一轮 LLM 的 `.tool` role content，脱敏后保留最多 16 KiB 前缀，超出时追加 `<truncated:N>`。
4. `AgentToolCallRunState.recordToolMessageContent` 现在读取保留内容后的 tool message，因此长 rate-limit body 仍能命中 `stopOnRateLimit`。
5. `StdioMCPProcessSession.matchesLaunchConfiguration(of:)` 显式比较启动输入，绕开 `MCPDescriptor ==` 的 id-only 身份语义；`StdioMCPClient.session(for:)` 在 start-in-flight 和 cached session 两个分支都能识别 stale session 并重启。
